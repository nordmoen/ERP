ERP: Evolutionary Algorithm

The evolutionary algorithm that is currently implented is a mu+lambda EA. In this EA, the next generation of individuals is combined with the previous generation of individuals and the worst individuals are deleted so that the population size remains constant. This type of evolutionary algorithm is very 'elitist' (it favors generating robots that resemble the best robots so far) and has the downside of prematurely converging. However, this EA is good enough to quickly get a robot to move in a certain direction.

The evolutionary algorithm works by calling a 'selection' (AKA offspring selection) and a 'replacement' (AKA survivor selection) operator. The selection operator selects new individuals that will need to be evaluated; it creates an offspring population. The replacement operator decides how to replace individuals in the parent population with individuals in the offspring population. The 'mutation' operator is called everytime after new offspring are generated. Since 'crossover' isn't implemented in yet, the variables that need to be set for the mutation operator are the controller mutation rate and the morphological mutation rate. By default these values are set to 0.1.

Using the ERP GUI, you can select different selection and replacement operators under the 'settings' -> 'EA' tab. You can also specify the mutation rate and parameters like the population size and the maximum number of generations the evolutionary algorithm should run.
